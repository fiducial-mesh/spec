# PCS Spec Changelog

## v0.1 (Draft) — 2026-05-16

Initial specification. Ten documents defining the complete governance
standard for PCS-governed plugin projects.

- 9 binding principles (verifiable/actionable, no thesis statements)
- Plugin project structure with discovery and compilation contracts
- Skill Spec (10 fields + acknowledged_external_risk)
- Runbook Spec (skill + judgment steps, inherited properties)
- Execution Profile (Node + Agent + Project Context, namespace normalization)
- Lifecycle states (draft/active/deprecated) with trust tiers (experimental→core)
- 5 validation gates (registration, pre/post-execution, promotion, compilation)
- Mutation Profile (pure/mutate_local/mutate_external) with idempotency (Option C)
- Failure modes with hand-off semantics and abort constraints
- Audit schema with CLCA process
- Compilation contract for Claude Code, Codex, Cline, Cursor

Reviewed through 12 CLCA cycles (Patton x3 structural, Einstein x3 focused).
Landscape validation: 8-agent sweep across Big-8 + startups + prior art.
